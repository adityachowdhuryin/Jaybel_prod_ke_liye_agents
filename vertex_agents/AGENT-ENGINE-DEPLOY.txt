Vertex AI Agent Engine - two-agent layout (Google ADK)
======================================================

Project: gls-training-486405
Region:  us-central1
Model:   gemini-2.5-flash

Folders (pass these paths to adk deploy agent_engine):
  vertex_agents/cost_metrics_agent
  vertex_agents/pa_orchestrator_agent

Deploy order
------------
1) Deploy cost_metrics_agent first. Configure DATABASE_URL for on-prem Postgres
   (tunnel + Secret Manager); do not commit secrets.
2) In Vertex / Agent Engine console, copy the HTTPS URL of the cost agent's
   Agent Card JSON (often ends with /.well-known/agent.json).
3) Set COST_AGENT_A2A_CARD_URL to that URL for the orchestrator and redeploy
   pa_orchestrator_agent.

Telemetry
---------
Use scripts/deploy-agent-engine.ps1 (includes):
  --trace_to_cloud --otel_to_cloud

PowerShell (from repo root)
---------------------------
  .\.venv\Scripts\Activate.ps1
  .\scripts\deploy-agent-engine.ps1 -Agent cost
  .\scripts\deploy-agent-engine.ps1 -Agent orchestrator

Phase 3 helper (legacy orchestrator path)
-----------------------------------------
If you run the FastAPI orchestrator service (`agents/orchestrator/main.py`),
you can configure Cloud Scheduler using:
  .\scripts\setup-proactive-scheduler.ps1

Local stack check
-----------------
After running .\scripts\start-all.ps1:
  .\scripts\check-local-stack.ps1

Next.js UI → Vertex Agent Engine (every chat invokes the engine)
---------------------------------------------------------------
The browser cannot call reasoningEngines:query with your user credentials. Use the
FastAPI orchestrator as a proxy:

1. Deploy cost_metrics_agent then pa_orchestrator_agent (order above).
2. gcloud auth application-default login  (same account that can invoke the engine)
3. Export GOOGLE_CLOUD_PROJECT, GOOGLE_CLOUD_LOCATION, and EITHER:
     ORCHESTRATOR_AGENT_ENGINE_RESOURCE=projects/.../reasoningEngines/ID
   OR the console "Copy query URL" value as:
     ORCHESTRATOR_AGENT_ENGINE_QUERY_URL=https://...reasoningEngines/ID:query
4. pip install -r agents/orchestrator/requirements.txt (includes google-cloud-aiplatform)
5. Run uvicorn on agents/orchestrator/main.py :8000
6. Keep frontend NEXT_PUBLIC_ORCHESTRATOR_URL=http://127.0.0.1:8000

GET /health shows agent_engine_chat_enabled: true when project + resource are set.
Agent Engine metrics in the console should increment when you send messages.
